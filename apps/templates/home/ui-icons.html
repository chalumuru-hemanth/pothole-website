Project Overview

This project aims to detect potholes using an ultrasonic sensor and GPS module, and then log the pothole coordinates to Google Sheets via a Wi-Fi connection. The system consists of an Arduino board, an ultrasonic sensor, a GPS module, and an L298N motor driver for controlling the movement of the vehicle.

Components

ESP 32

A powerful Wi-Fi and Bluetooth-enabled microcontroller.

L298N Motor Driver

Controls the movement of the vehicle in response to pothole detection (optional).

NEO-6M GPS Module

Provides accurate location coordinates for logging pothole data.

NEO-6M GPS Module

Detect the presence of potholes on the road.

Functionality

  1. Sensor Data Acquisition:
    • The Arduino board reads data from the ultrasonic sensor to detect the presence of potholes on the road.
    • Additionally, it reads GPS data from the NEO-6M module to obtain the current location coordinates.
  2. Wi-Fi Connection:
    • The system establishes a Wi-Fi connection using the ESP32 module to communicate with Google Sheets.
  3. Pothole Detection:
    • The ultrasonic sensor measures the distance to the ground to detect the presence of potholes.
    • If a pothole is detected (distance exceeds a threshold), the system proceeds to log the GPS coordinates.
  4. Data Logging:
    • Once a pothole is detected, the system constructs a URL containing the timestamp, GPS coordinates, and distance.
    • It then sends an HTTP GET request to a Google Apps Script web app, which logs the data to a Google Sheets spreadsheet.
  5. Motor Control (Optional):
    • The system includes motor control functions to maneuver the vehicle in response to detected potholes. However, this functionality is not utilized in the current implementation.
